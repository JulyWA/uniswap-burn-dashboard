📖 名词与机制说明(点击展开)

① 价值捕获 = 回购归集(无销毁)

  • 无销毁:AAVE 固定 16,000,000,无增发无销毁。价值捕获靠回购——DAO 用协议收入经 CoW 买回 AAVE 归 DAO(安全模块激励等再分配),币没销毁、只从流通转到 DAO 手里。
    注:买回的 AAVE 会打进 DAO 的一个地址,历史上是 Collector(0x464C…);而生态储备(0x25F2…)那 ~59万 AAVE 是 DAO 早期就有的存量、不是回购买来的(别当成回购累积)。
  • 预算 $30M/年(2026-03 从 $50M 削减):削减原因=借款利息较峰值降 ~25%,且 DAO 总支出 $190M > 2025 收入 $142M(赤字),砍回购拉回可持续。"计划回购额"= 年预算/52,是计划值。
  • ⚠️ 计划 vs 链上实际(关键):链上实际自 2026-04-19 暂停——4月18 rsETH 桥事故触发治理暂停(ARFC 24686);Aavenomics 3.0 新自动化机制 6-29 宣布上线、链上落点待确认(三源:链上+新闻+推特)。所以回本/收益率/覆盖率的"计划"值目前是理论值,链上实际 ≈ 0 → 暂停期以「按协议收入」口径看回本;恢复后自动跟上。

② 数字口径(容易搞混的)

  • 总手续费 ≠ 协议收入:总手续费=借款人付的全部利息;协议收入=其中 DAO 抽走的一小块(抽成率 ~13%),其余 ~87% 给存款人。
  • 净 TVL vs 总存款:净流动性 TVL=存款−借款(看规模);总存款=净TVL+借款,是利用率、平均利率的分母。
  • 回购覆盖率:=回购 ÷ DAO 净收入,看回购可不可持续(<100%净利够 / >100%靠金库补)。但净收入只扣服务商 $20M、没扣安全/流动性激励(总支出 $190M),故 71% 偏乐观。
  • GHO 收入:GHO 无存款侧、利息几乎全归 DAO。链上直读借款利率×借款量,是主网口径、为下界(漏 L2/GSM 铸造的 GHO)。holdersRevenue 对 Aave=0。
  • 分给团队/服务商:=DAO 付服务商 $20M/年(TokenLogic 口径,工资+grants)。"团队赚多少"含 Avara 私有公司利润不可知;团队持仓也在未标注钱包、链上查不到(故不列)。

③ 质押 / 安全模块(分两层)

  • Umbrella 是什么、干嘛的:Aave 的坏账保险。用户质押 aToken(aUSDC 等)或 GHO 进金库赚激励;协议一旦出坏账,这些质押资产被 slash 罚没去补窟窿。规模越大抗风险越强。
  • 两层:① AAVE 计价——stkAAVE 锁的 AAVE(约总量 13.5%),"承诺给安全"的对齐指标,仍算流通、不从流通扣;② 非 AAVE 安全垫——stkGHO + Umbrella 金库。⚠️ Umbrella 未完整:仅纳入已验证地址的金库(当前只 USDC),USDT/WETH 待补,真实安全垫更大。

④ 数据来源与交叉强度(核对可信度)

  • 🟢 供给侧(强):总量 16M = 链上+CoinGecko+常量三源;流通 = CoinGecko⟷链上(总量−储备)双源;价格 = CoinGecko⟷Coinbase 独立双源(±2%)。
  • 🟢 链上直读(权威):储备/stkAAVE/stkGHO/Umbrella 持仓、GHO 借款利率——链上不会"错",地址已人工验。
  • 🟢 TVL 已补链上独立交叉:DefiLlama ⟷ 链上聚合(Aave Oracle 批量价 × 各 reserve aToken/债务)主网净TVL,±15% 核对(实测差 ~2%)。真独立源(链上 vs 聚合器)。
  • 🔴 收入侧仍单源(残余缺口):手续费 / 协议收入 / 未偿借款 / 多链分布 仍只来自 DefiLlama。回购覆盖率/收益率/抽成率等衍生量追溯到它 → DefiLlama 若整体错、内部自洽仍通过。补独立源需 Token Terminal/Dune(需 key)或链上追 Collector 入账(工程量大),尚未接入。
  • 🟡 静态参数:回购 $30M / DAO 支出 $20M / $190M ——录入时多源核过,之后不再持续交叉;GOV_FORUM 命中相关提案会提示更新。
